@schema.prisma 파일에 자산 정보를 관리하는 DB Schema를 만들려고 해.
아래 요구사항을 정리해서 혼동을 일으키는 용어가 있다면 더 좋은 표현으로 다듬어 주고,
테이블을 어떻게 만들고 데이터 입력을 받아서 관리하는 게 좋을지 추천해줘.

## 계좌 관리
- 원화 잔고, 외화 잔고 등을 관리함
- 계좌에서 특정 상품(자산)을 매수함 (S&P 500 등)
- 계좌의 종류 : 은행 계좌 (예금, 적금), 증권사 계좌 (일반, CMA, 연금저축, IRP, ISA 등)
- 사용자

## 자산 관리
- 자산의 개당 가격, 수량을 기록하며, 외화 자산의 경우 환율 및 기준 통화 가격까지 관리함
  (원화 기준으로 가치 추적이 가능하도록)
- 기존 데이터는 특정 시점 보유 개수와 환율, 가격만 존재함
  (snapshot 형태로, 매수/매도 시점 가격 및 환율 없음)
- 향후 입력될 데이터는 매수/매도 개수 및 가격, 수량, 환율을 기록할 예정
- 자산에는 메타데이터를 추가하여 조회 시 그룹화할 예정
  - 위험도 (주식: 위험자산, 채권: 안전자산 등)
  - 대분류 (주식,예금,금,채권)
  - 소분류 (원화예금, 성장주식, 배당주식 등)
  - 거래처 (농협은행, 나무증권, 메리츠증권 등)
  - 계좌명 : 자산을 구매한 연관 계좌
    (계좌 단위 조회 예시: 나무증권 IRP에 보유중인 자산 목록 조회)
  - 향후 확장될 수 있으므로, 매번 컬럼을 만들 지 아니면 json object 형태의 관리가 좋을지 고민
- 매수/매도가 없어도 주기적으로 (1주일 등) 원화 기준 자산 가치를 추적할 수 있어야 함
  (가격 정보는 향후 API 등을 통해 연동)

## 자산 및 계좌 조회
- 계좌 잔고와 자산 가격을 합쳐 월별, 연별 추이를 확인할 수 있어야 함
- 전체 사용자 및 개별 사용자 단위 자산 추이
